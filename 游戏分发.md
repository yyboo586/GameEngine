# 游戏分发平台
- 这是一个游戏分发平台，支持游戏APK、H5包上传，游戏分类管理，用户收藏、预约等功能。

## 功能列表
### 游戏推荐
- 先按照Tag相似度匹配
    - 取目标游戏的所有Tag，找出"拥有任一相同Tag的其他游戏"，按"匹配到的相同Tag数量"降序排列。
    - 相同评分时，用 总评分、评分次数、下载次数、创建事件 作为热度兜底排序。
- 数量不足时回退到相同分类
- 再不足时用全站热门补齐
- 过滤
    - 排除自身
    - 过滤下架/未发布




### 榜单逻辑

#### 热门榜逻辑（已实现）
- **排序公式**: 下载量×0.5 + 收藏数×0.3 + 评分×0.2
- **数据来源**: 基于现有数据库字段 `download_count`、`favorite_count`、`rating_score`
- **筛选条件**: 只显示已上架的游戏（status = 2）
- **分页支持**: 支持分页查询，默认每页10条
- **接口路径**: `GET /api/v1/game-engine/games/ranking/hot`

#### 新游榜逻辑（已实现）
- **时间范围**: 最近30天内发布的游戏
- **排序规则**: 按发布时间倒序排列
- **数据来源**: 基于现有数据库字段 `publish_time`
- **筛选条件**: 只显示已上架的游戏（status = 2）
- **分页支持**: 支持分页查询，默认每页10条
- **接口路径**: `GET /api/v1/game-engine/games/ranking/new`

#### 技术实现特点
- **简单有效**: 只使用现有数据字段，无需复杂计算
- **性能优化**: 利用数据库索引进行高效查询
- **实时更新**: 榜单数据实时计算，无需定时任务
- **扩展性好**: 后续可以轻松添加更多榜单类型


### 用户行为记录（已实现）

#### 业务逻辑变更
- **用户登录要求**: 用户必须登录后才能玩游戏，确保行为记录精准性
- **移除浏览器指纹**: 不再需要匿名用户识别，简化系统架构
- **专注用户体验**: 专注于登录用户的搜索历史和游戏行为记录

#### 搜索历史管理（已实现）
- **数据存储**: 搜索历史表 `t_search_history`
- **主要字段**: 用户ID、搜索关键词、搜索时间、结果数量
- **功能特性**: 
  - 记录用户搜索行为
  - 支持清空搜索历史
  - 提供热门搜索统计
  - 按时间倒序排列

#### 游戏行为记录（已实现）
- **数据存储**: 游戏行为记录表 `t_game_behavior`
- **主要字段**: 用户ID、游戏ID、行为类型、行为时间、IP地址
- **行为类型**: 查看、下载、收藏、评分
- **功能特性**: 完整记录用户游戏行为轨迹

#### 已实现功能
1. **搜索历史服务**: 完整的搜索历史记录和管理
2. **游戏行为记录**: 记录用户的各种游戏行为
3. **热门搜索统计**: 统计最近7天的热门搜索关键词
4. **历史数据查询**: 支持按用户查询历史记录
5. **搜索历史清空**: 用户可清空个人搜索历史

#### 技术特点
- **简单高效**: 基于用户ID的精准记录，无需复杂识别逻辑
- **性能优化**: 合理的数据库索引设计，支持高效查询
- **数据安全**: 用户只能访问自己的数据，确保隐私安全
- **易于维护**: 清晰的业务逻辑，便于后续功能扩展

#### 使用方式
```go
// 记录搜索历史
service.DeviceFingerprint().AddSearchHistory(ctx, userID, keyword, resultCount)

// 记录游戏行为
service.DeviceFingerprint().RecordGameBehavior(ctx, userID, gameID, behaviorType, ipAddress)

// 获取搜索历史
service.DeviceFingerprint().GetSearchHistory(ctx, userID, limit)

// 清空搜索历史
service.DeviceFingerprint().ClearSearchHistory(ctx, userID)
```
