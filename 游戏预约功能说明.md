# 游戏预约功能说明

## 功能概述

本系统实现了游戏预约功能，包括：

1. **即将上新功能**：获取未上架的游戏列表，按创建时间逆序排列
2. **游戏预约功能**：用户只能预约未上架的游戏
3. **预约管理功能**：用户可以查看、取消预约

## 数据库表结构

### 游戏表 (t_game) - 新增字段
- `status`: 游戏状态 (1:未上架 2:已上架 3:已下架)

### 游戏预约表 (t_game_reserve)
- `id`: 主键
- `game_id`: 游戏ID
- `user_id`: 用户ID
- `create_time`: 创建时间
- `update_time`: 更新时间

### 游戏预约接口

#### 1. 预约游戏
- **接口**: `POST /api/v1/game-engine/games/{id}/reserve`
- **参数**: 
  - `id`: 游戏ID (路径参数)
  - `user_id`: 用户ID (请求体)
- **响应**: `{"success": true}`

#### 2. 取消游戏预约
- **接口**: `DELETE /api/v1/game-engine/games/{id}/reserve`
- **参数**: 
  - `id`: 游戏ID (路径参数)
  - `user_id`: 用户ID (请求体)
- **响应**: `{"success": true}`

#### 3. 查询游戏预约状态
- **接口**: `GET /api/v1/game-engine/games/{id}/reserve?user_id={user_id}`
- **参数**: 
  - `id`: 游戏ID (路径参数)
  - `user_id`: 用户ID (查询参数)
- **响应**: `{"is_reserved": true}`

## 业务逻辑

### 即将上新逻辑
1. 查询状态为"未上架"的游戏
2. 按创建时间逆序排列
3. 支持分页查询
4. 返回完整的游戏信息（包括分类、标签、媒体信息）

### 本月上新逻辑
1. 查询状态为"已上架"的游戏
2. 筛选发布时间在当前月份内的游戏
3. 按发布时间逆序排列
4. 支持分页查询
5. 返回完整的游戏信息（包括分类、标签、媒体信息）

### 游戏预约逻辑
1. 检查游戏是否存在
2. 验证游戏状态是否为"未上架"
3. 检查用户是否已经预约过该游戏
4. 如果未预约，则添加预约记录
5. 如果已预约，则直接返回成功

### 预约限制
- 只能预约状态为"未上架"的游戏
- 每个用户对同一游戏只能有一个预约记录
- 游戏上架后，预约记录仍然保留（可用于后续通知）

## 使用示例

### 获取即将上新的游戏
```bash
curl -X GET "http://localhost:8000/api/v1/game-engine/games/upcoming?page=1&size=10" \
  -H "Authorization: Bearer {token}"
```

### 获取本月上新的游戏
```bash
curl -X GET "http://localhost:8000/api/v1/game-engine/games/new/this-month?page=1&size=10" \
  -H "Authorization: Bearer {token}"
```

### 预约游戏
```bash
curl -X POST "http://localhost:8000/api/v1/game-engine/games/1/reserve" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer {token}" \
  -d '{"user_id": 123}'
```

### 取消预约
```bash
curl -X DELETE "http://localhost:8000/api/v1/game-engine/games/1/reserve" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer {token}" \
  -d '{"user_id": 123}'
```

### 查询预约状态
```bash
curl -X GET "http://localhost:8000/api/v1/game-engine/games/1/reserve?user_id=123" \
  -H "Authorization: Bearer {token}"
```

### 获取用户预约列表
```bash
curl -X GET "http://localhost:8000/api/v1/game-engine/users/123/reserves?page=1&size=10" \
  -H "Authorization: Bearer {token}"
```

## 注意事项

1. 用户ID需要在实际项目中与用户认证系统集成
2. 游戏状态变更需要管理员权限
3. 预约功能仅对未上架游戏开放
4. 所有操作都包含适当的错误处理和日志记录
5. 建议对预约接口添加频率限制，防止恶意预约
6. 本月上新接口会自动计算当前月份，无需手动指定时间范围
