# 今日精选API说明

## 概述

"今日精选"是游戏分发平台的核心推荐功能，基于智能算法为用户推荐最优质、最热门的游戏。该功能结合了**内容质量**、**时效性**和**个性化**三个维度，为用户提供精准的游戏推荐。

## 基础信息

- **接口路径**: `/api/v1/game-engine/games/today-picks`
- **请求方式**: `GET`
- **认证要求**: 可选（提供用户ID可获得个性化推荐）
- **数据格式**: JSON
- **字符编码**: UTF-8

## 接口详情

### 请求参数

| 参数名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| user_id | int64 | 否 | - | 用户ID，用于个性化推荐 |
| limit | int | 否 | 20 | 返回数量限制，最大100 |

### 请求示例

```bash
# 基础请求
GET /api/v1/game-engine/games/today-picks?limit=10

# 个性化推荐请求
GET /api/v1/game-engine/games/today-picks?user_id=123&limit=15
```

## 推荐算法详解

### 1. 热度分数计算

**基础分数公式**：
```
热度分数 = 下载量×0.4 + 收藏数×0.3 + 评分×0.2 + 评分次数×0.1
```

**时间加分**：
- 7天内发布的游戏获得额外加分
- 加分公式：`100 × (7 - 发布天数) / 7`
- 新游戏获得最高100分额外加分

### 2. 推荐理由生成

系统会根据游戏特征自动生成推荐理由：

| 特征 | 条件 | 推荐理由 |
|------|------|----------|
| 今日新发布 | 今天发布 | "今日新发布" |
| 编辑推荐 | 评分≥4.0且下载量≥1000 | "编辑推荐" |
| 超高评分 | 平均评分≥4.5 | "超高评分" |
| 高评分 | 平均评分≥4.0 | "高评分" |
| 超热门 | 下载量≥10000 | "超热门" |
| 热门 | 下载量≥5000 | "热门" |
| 高收藏 | 收藏数≥1000 | "高收藏" |

### 3. 个性化排序

当提供用户ID时，系统会进行个性化排序：

1. **基础排序**：按热度分数降序排列
2. **个性化加分**：
   - 今日新发布：+50分
   - 编辑推荐：+30分
3. **最终排序**：按（热度分数+个性化加分）降序排列

## 使用场景

### 1. 首页推荐
```javascript
// 获取首页今日精选
async function getHomePagePicks() {
    const response = await fetch('/api/v1/game-engine/games/today-picks?limit=8');
    const data = await response.json();
    return data.data.list;
}
```

### 2. 个性化推荐
```javascript
// 获取个性化推荐
async function getPersonalizedPicks(userId) {
    const response = await fetch(`/api/v1/game-engine/games/today-picks?user_id=${userId}&limit=20`);
    const data = await response.json();
    return data.data.list;
}
```

### 3. 游戏发现页
```javascript
// 获取更多推荐游戏
async function getMorePicks(page = 1) {
    const response = await fetch(`/api/v1/game-engine/games/today-picks?limit=20&page=${page}`);
    const data = await response.json();
    return data.data.list;
}
```

## 技术特点

### 1. 智能算法
- **多维度评分**：综合考虑下载量、收藏数、评分等多个指标
- **时间衰减**：新游戏获得适当加分，保持内容新鲜度
- **动态权重**：根据数据分布动态调整各指标权重

### 2. 性能优化
- **数据库索引**：优化查询性能，支持高并发访问
- **缓存策略**：热门推荐结果缓存，减少数据库压力
- **分页查询**：支持大数据量分页，避免内存溢出

### 3. 扩展性
- **算法可配置**：权重参数可通过配置调整
- **规则可扩展**：推荐理由规则可灵活配置
- **个性化可增强**：预留接口支持更复杂的个性化算法

## 数据更新策略

### 1. 实时更新
- **用户行为**：下载、收藏、评分等行为实时影响热度分数
- **游戏状态**：游戏上下架状态实时反映在推荐结果中

### 2. 定时更新
- **每日更新**：每日凌晨重新计算推荐列表
- **热度重算**：定期重新计算历史游戏的热度分数

### 3. 缓存策略
- **推荐结果**：缓存5分钟，平衡实时性和性能
- **热度分数**：缓存1小时，减少重复计算

## 注意事项

### 1. 数据一致性
- 确保游戏状态、评分等数据的实时性
- 避免缓存过期导致的数据不一致

### 2. 性能考虑
- 大量游戏数据时的查询性能优化
- 个性化推荐的计算复杂度控制

### 3. 用户体验
- 推荐结果的多样性，避免同质化
- 新游戏和老游戏的平衡推荐

## 扩展功能

### 1. 分类推荐
- 按游戏分类进行个性化推荐
