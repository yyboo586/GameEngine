# 游戏收藏和评分功能说明

## 功能概述

本系统实现了游戏收藏和评分功能，包括：

1. **游戏收藏功能**：用户可以收藏/取消收藏游戏
2. **游戏评分功能**：用户可以对游戏进行1-5星评分
3. **评分统计功能**：自动计算游戏的平均评分和评分次数

## 数据库表结构

### 游戏收藏表 (t_game_favorite)
- `id`: 主键
- `game_id`: 游戏ID
- `user_id`: 用户ID
- `create_time`: 创建时间
- `update_time`: 更新时间

### 游戏评分表 (t_game_rating)
- `id`: 主键
- `game_id`: 游戏ID
- `user_id`: 用户ID
- `rating`: 评分(1-5)
- `create_time`: 创建时间
- `update_time`: 更新时间

## HTTP接口

### 游戏收藏接口

#### 1. 添加游戏收藏
- **接口**: `POST /games/{id}/favorite`
- **参数**: 
  - `id`: 游戏ID (路径参数)
  - `user_id`: 用户ID (请求体)
- **响应**: `{"success": true}`

#### 2. 取消游戏收藏
- **接口**: `DELETE /games/{id}/favorite`
- **参数**: 
  - `id`: 游戏ID (路径参数)
  - `user_id`: 用户ID (请求体)
- **响应**: `{"success": true}`

#### 3. 获取游戏收藏状态
- **接口**: `GET /games/{id}/favorite?user_id={user_id}`
- **参数**: 
  - `id`: 游戏ID (路径参数)
  - `user_id`: 用户ID (查询参数)
- **响应**: `{"is_favorite": true}`

### 游戏评分接口

#### 1. 添加游戏评分
- **接口**: `POST /games/{id}/rating`
- **参数**: 
  - `id`: 游戏ID (路径参数)
  - `user_id`: 用户ID (请求体)
  - `rating`: 评分(1-5) (请求体)
- **响应**: `{"success": true}`

#### 2. 获取游戏评分
- **接口**: `GET /games/{id}/rating`
- **参数**: 
  - `id`: 游戏ID (路径参数)
- **响应**: `{"rating": 4, "rating_count": 10}`

## 业务逻辑

### 游戏收藏逻辑
1. 检查游戏是否存在
2. 检查用户是否已经收藏过该游戏
3. 如果未收藏，则添加收藏记录并更新游戏收藏次数
4. 如果已收藏，则直接返回成功

### 游戏评分逻辑
1. 检查游戏是否存在
2. 验证评分是否在1-5范围内
3. 检查用户是否已经评分过
4. 如果已评分，则更新现有评分
5. 如果未评分，则添加新评分
6. 自动更新游戏的评分统计（总评分和评分次数）

### 评分计算逻辑
- 平均评分 = (总评分 + 评分次数/2) / 评分次数
- 使用四舍五入算法计算最终评分

## 使用示例

### 收藏游戏
```bash
curl -X POST "http://localhost:8000/games/1/favorite" \
  -H "Content-Type: application/json" \
  -d '{"user_id": 123}'
```

### 取消收藏
```bash
curl -X DELETE "http://localhost:8000/games/1/favorite" \
  -H "Content-Type: application/json" \
  -d '{"user_id": 123}'
```

### 评分游戏
```bash
curl -X POST "http://localhost:8000/games/1/rating" \
  -H "Content-Type: application/json" \
  -d '{"user_id": 123, "rating": 5}'
```

### 查询评分
```bash
curl "http://localhost:8000/games/1/rating"
```

## 注意事项

1. 用户ID需要在实际项目中与用户认证系统集成
2. 评分范围限制为1-5星
3. 每个用户对同一游戏只能有一个评分记录
4. 收藏和评分操作都会自动更新游戏表的统计字段
5. 所有操作都包含适当的错误处理和日志记录
